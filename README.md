## 🌟 项目简介

**Novel AI Agent** 是一个基于 LLM（大语言模型）的结构化小说创作辅助系统。它采用**多阶段渐进创作**的理念，从世界观设定到正文写作，逐层细化，让AI成为你的创作搭档，而非简单的文本生成器。

### 核心理念

- **🎯 AI 先行，用户审校**：每个阶段 AI 自动生成草稿，用户在界面上直接编辑修改，确认后才进入下一阶段
- **📖 结构化创作**：从小说预设 → 大纲 → 章节大纲 → 小节内容，层层递进
- **🧠 记忆与一致性**：通过"小说预设"系统维护人物、势力、地点等设定的一致性
- **✍️ 续写支持**：支持导入已有小说进行智能续写，自动分析前作世界观和伏笔

## 🚀 快速开始

### 1. 环境准备

确保你的系统已安装：
- **Node.js** 18+ 
- **npm** 或 **yarn**

### 2. 安装依赖

```bash
cd my-app
npm install
```

### 3. 配置 API Key（⚠️ 重要！）

**复制环境变量模板：**

```bash
cp .env.example .env.local
```

**编辑 `.env.local` 文件，配置你的 API 信息：**

#### 方案一：使用 Anthropic Claude（推荐）

```env
ANTHROPIC_API_KEY=sk-ant-api03-你的API密钥
```

#### 方案二：使用 OpenAI 兼容格式（支持 DeepSeek、Moonshot 等）

```env
# DeepSeek
OPENAI_API_KEY=sk-你的DeepSeek密钥
OPENAI_BASE_URL=https://api.deepseek.com/v1

# 或 Moonshot (Kimi)
OPENAI_API_KEY=sk-你的Moonshot密钥
OPENAI_BASE_URL=https://api.moonshot.cn/v1

# 或其他任意 OpenAI 兼容 API
OPENAI_API_KEY=你的密钥
OPENAI_BASE_URL=https://your-api-endpoint.com/v1
```

> 💡 **提示**：可以同时配置多个 API，系统会根据模型名称自动选择合适的 API 调用。

### 4. 启动应用

```bash
npm run dev
```

访问 http://localhost:3000 开始使用！

## 📖 使用指南

### 🏠 首页与项目管理

进入应用后，你会看到项目列表页面：

- **新建项目**：从零开始创作一部新小说
- **导入续写**：导入已有小说文件进行续写
- **项目卡片**：显示项目类型（原创/续写）、当前阶段、最后更新时间

### 📝 五阶段创作流程

#### Stage 1: 小说预设（Novel Preset）

这是创作的地基，需要填写：

| 模块 | 说明 | 示例字段 |
|------|------|----------|
| **基本信息** | 小说的整体定位 | 标题、题材、基调、目标字数、简介、核心主题 |
| **人物设定** | 所有登场角色的详细档案 | 姓名、角色定位、外貌、性格、背景故事、能力、动机、人物弧线 |
| **势力设定** | 世界观中的组织/阵营 | 宗门、国家、家族等的立场、目标、内部结构、相互关系 |
| **地点设定** | 重要场景/副本 | 秘境、城市、战场等的地理位置、特色、危险、资源 |
| **高潮场景** | 关键剧情节点规划 | 战斗高潮、情感高潮、剧情反转等详细描述 |

**💡 小贴士**：
- 每个字段都可以点击"AI 补全"按钮，让 AI 基于已有信息生成建议
- 也可以只写一句话主题，让 AI 一键生成完整的建议

#### Stage 2: 大纲规划（Outline）

点击"AI 生成大纲"，系统会基于小说预设自动生成：

- 📖 **故事主线**：起承转合的整体脉络
- 📚 **卷划分**：3-5 卷的结构规划
- 📅 **每卷核心事件**：该卷要发生的关键剧情
- ⏰ **时间线**：重要事件的时间顺序

**你可以**：
- 编辑任意卷的标题、核心事件、字数目标
- 调整章节分布
- 重新生成不满意的部分

#### Stage 3: 章节大纲（Chapter Outlines）

选择要生成的卷，AI 会为每一章生成：

- 章节标题和摘要
- 关键事件列表
- 开头钩子（吸引读者）
- 结尾悬念（保持期待）
- 伏笔埋下/回收点
- 人物状态变化

#### Stage 4: 小节列表（Sections）

将每章细分为若干小节（每节约 500-1500 字）：

- 场景时间和地点
- 叙述视角（POV）
- 内容概要
- 情感节拍
- 写作要点提示

#### Stage 5: 正文写作（Writing）

核心创作阶段！点击"生成正文"，AI 会：

1. 读取小说预设（确保人物、设定一致）
2. 读取前一节的结尾（确保衔接自然）
3. 读取本章大纲（确保不偏离主线）
4. **流式生成**正文内容

**写作界面功能**：
- ✏️ 实时编辑生成的内容
- 🔄 重新生成当前小节
- ✅ 确认并进入下一个小节
- 📊 实时字数统计

## 🔄 续写功能（Continuation）

想要为自己的旧作写续集？或基于已有小说继续创作？使用**续写模式**！

### 如何开始续写

**方式一：直接导入（推荐）**
1. 点击首页"导入续写"按钮
2. 输入续写作品名称和原作标题
3. 上传原作文件（支持 `.txt`、`.md`，UTF-8 编码）
4. AI 自动分析并帮助续写

**方式二：从已完成项目续写**
1. 找到已完成的原创项目
2. 点击"续写"按钮
3. 自动创建续写项目并关联原作

### 续写流程

```
输入文件 → AI 分析 → 圣经同步 → 续写大纲 → 章节规划 → 正文写作
```

**AI 分析内容包括**：
- 🌍 世界观总结
- 👥 角色信息（主角、配角、关系）
- 🔮 未回收的伏笔
- 📜 已发生的主要事件
- 🏛️ 势力/组织
- 📍 关键地点

## 🧠 记忆与一致性机制

### 什么是"小说预设"？

**小说预设（Novel Preset）** 是本系统的核心创新。它是整部小说的"设定百科全书"，包含所有人物、势力、地点、伏笔等关键信息。

### 记忆如何工作？

#### 1. 上下文注入（Context Injection）

每次 AI 生成内容时，系统会自动将相关信息注入 System Prompt：

```
┌─────────────────────────────────────────────────────┐
│  System Prompt                                      │
├─────────────────────────────────────────────────────┤
│  【小说基本信息】                                    │
│  标题、题材、基调、核心主题...                        │
│                                                      │
│  【相关人物设定】 ← 动态提取本节涉及的角色            │
│  主角：性格、能力、当前状态、动机...                  │
│  配角A：...                                          │
│                                                      │
│  【相关势力设定】                                    │
│  青云宗：立场、目标、与主角关系...                    │
│                                                      │
│  【相关地点设定】                                    │
│  迷雾森林：特色、危险、资源...                        │
│                                                      │
│  【前一节结尾】 ← 保证衔接自然                        │
│  "......最后三段内容"                                │
│                                                      │
│  【本节任务】                                        │
│  场景、情感节拍、内容概要...                          │
└─────────────────────────────────────────────────────┘
```

#### 2. 人物状态追踪

每写完一小节，系统会自动分析并更新人物状态：

```typescript
// 示例：人物状态变化记录
{
  characterId: " protagonist_01",
  changeDescription: "在战斗中受伤，右臂骨折；获得神秘老人的认可"
}
```

这些状态会被记录，并在后续章节中自动注入 Prompt。

#### 3. 伏笔追踪系统

系统维护一个伏笔数据库：

```typescript
interface PlotThread {
  id: string;
  description: string;           // 伏笔内容
  plantedInSection: string;      // 在哪个小节埋下
  resolvedInSection?: string;    // 在哪个小节回收
  status: "planted" | "resolved" | "abandoned";
}
```

在生成章节大纲时，AI 会参考此数据库，确保伏笔得到合理回收。

#### 4. 名词一致性检查

维护小说专有名词词典，自动检查同一概念的名称是否一致（如"青云宗"不会时而写成"清云宗"）。

### 记忆等级设置（AI Memory Level）

为了更好地满足专业小说创作的需求，系统提供了 **5 级记忆等级**，让你可以精确控制 AI 在创作时注入的上下文详细程度。不再为了节省 Token 而牺牲创作质量！

#### 如何设置

在「小说预设」→「基本信息」页面底部，找到 **AI 记忆等级** 卡片进行设置。

#### 五级记忆详解

| 等级 | 名称 | 注入内容 | 适用场景 |
|:----:|------|----------|----------|
| **1** | 🌱 基础记忆 | • 当前步骤必需的最少信息<br>• 当前小节相关人物简介 | 快速测试、短篇创作、对成本极敏感的场景 |
| **2** | 📚 标准记忆 | • 核心设定概要<br>• 上一节结尾内容<br>• 本节涉及人物 | 平衡质量与成本的一般性创作 |
| **3** | ✨ 增强记忆 ⭐推荐 | • **完整小说预设**<br>• 当前阶段相关上下文<br>• 主要人物 + 当前状态 | **推荐设置**，确保良好的连贯性和一致性 |
| **4** | 🎯 完整记忆 | • 完整圣经<br>• **完整大纲规划** + 卷详情<br>• 前序章节摘要<br>• 全部人物状态追踪 | 长篇创作、多卷小说、需要深度上下文理解 |
| **5** | 👑 最强记忆 | • **全部可用信息**：圣经、大纲、前序章节<br>• **伏笔追踪数据库**<br>• 完整写作要点<br>• 极致一致性保障 | 追求极致一致性的复杂剧情、多卷长篇小说 |

#### 各阶段注入内容对照

| 内容类型 | L1 基础 | L2 标准 | L3 增强 | L4 完整 | L5 最强 |
|----------|:-------:|:-------:|:-------:|:-------:|:-------:|
| 小说预设 | 仅当前 | 核心概要 | ✅ 完整 | ✅ 完整 | ✅ 完整 |
| 大纲规划 | ❌ | 仅当前卷 | 主线+卷概览 | ✅ 完整+详情 | ✅ 完整+详情 |
| 章节信息 | ❌ | ❌ | 当前章 | 前3节+本章 | 前5节+本章+前卷 |
| 人物追踪 | 仅本节 | 本节人物 | 主要+状态 | ✅ 全部+状态 | ✅ 全部+状态 |
| 伏笔追踪 | ❌ | ❌ | ❌ | 相关提醒 | ✅ 完整数据库 |
| 写作要点 | 仅当前 | 仅当前 | 仅当前 | ✅ 完整 | ✅ 完整 |

#### 使用建议

- **一般创作（默认）**：选择 **3-增强记忆**，在保证质量的同时控制成本
- **长篇小说**：选择 **4-完整记忆** 或 **5-最强记忆**，确保跨章节的连贯性
- **短篇/测试**：选择 **1-基础记忆** 或 **2-标准记忆**，快速验证创意
- **关键章节**：临时切换到 **5-最强记忆**，确保重要情节的一致性

> 💡 **专业提示**：记忆等级可以在创作过程中随时调整。对于日常写作使用 L3，遇到关键转折点或复杂剧情时切换到 L4/L5，完成后再调回 L3。

## ⚙️ 环境变量详细说明

### API 配置（必需）

| 变量名 | 说明 | 必需 |
|--------|------|------|
| `ANTHROPIC_API_KEY` | Anthropic Claude API 密钥 | 使用 Claude 时 |
| `OPENAI_API_KEY` | OpenAI 格式 API 密钥 | 使用 OpenAI 兼容 API 时 |
| `OPENAI_BASE_URL` | OpenAI 格式 API 地址 | 使用第三方 API 时 |

### 模型配置（可选）

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `SETTING_MODEL` | 设定模型（大纲、人物等） | `gpt-4-turbo` |
| `WRITING_MODEL` | 写作模型（正文生成） | `gpt-4-turbo` |

**建议配置**：
- **设定模型**：使用较强的模型（如 GPT-4、Claude Sonnet/Opus）
- **写作模型**：可以使用轻量模型（如 GPT-3.5、Claude Haiku）节省成本

## 📝 最佳实践

### 1. 小说预设填写技巧

- **不必一次性填完**：先填核心信息，后续随时补充
- **利用 AI 补全**：先写一句话描述，让 AI 展开详细设定
- **保持动态更新**：写作过程中发现新设定，及时添加到圣经

### 2. 大纲调整建议

- **先生成再调整**：让 AI 生成初稿，再人工调整细节
- **关注节奏**：确保高潮场景分布均匀，避免前松后紧
- **预留空间**：为灵感迸发预留一些自由发挥的空间

### 3. 正文写作技巧

- **控制小节长度**：500-1500 字为宜，太长容易偏离概要
- **及时确认**：满意的内容及时确认，进入下一个小节
- **利用重新生成**：对不满意的部分，可以重新生成或局部修改

### 4. 续写功能使用建议

- **完整上传原作**：上传完整原作以获得最佳分析效果
- **检查分析结果**：AI 分析后，务必检查并修正提取的信息
- **保持文风一致**：在圣经中记录原作文风特点，指导续写

## ⚠️ 注意事项

1. **API 费用**：AI 生成功能需要消耗 API Token，请留意费用
2. **数据隐私**：输入文件存储在本地，AI 分析时只传输必要内容
3. **版权合规**：仅续写自己拥有版权或获得授权的作品
4. **上下文限制**：大模型有上下文限制，超长的前文可能需要精简注入
